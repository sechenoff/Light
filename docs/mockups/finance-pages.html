Логика секции

  • Вся секция — только для руководителя. Кладовщик и техник не видят /finance ни в меню, ни по прямой ссылке (403). В roles-permissions зафиксируем.
  • Страницы отвечают на строго четыре вопроса. Левый цветной стрип на KPI: rose — долги, amber — ожидания, emerald — выручка, slate — расходы. Одна и та же палитра повторяется на таблицах и календаре — глаз привыкает.
  • Расчёт долга — существующий computeDebts() в apps/api/src/services/finance.ts. Докручиваем aging-бакеты (>30 / 1–30 / 1–7 дней / в графике) и сортировку по возрасту задолженности.
  • Ожидания и поступления — вытягиваем из поля dueDate брони и отдельной таблицы Payment (добавить в Prisma: id, bookingId, amount, receivedAt, method). Календарь — это агрегация платежей по дню.
  • Расходы — новая таблица Expense (id, date, categoryEnum, amount, description, documentUrl?, linkedBookingId?). Пять категорий фиксированно; «Прочее» — открытый текст.
  • Связь с ремонтом. Если в repair-workflow создаётся карточка ремонта с платной работой — автоматически предлагается создать расход категории «Ремонт» с привязкой к единице и броне. Одно действие, не двойной ввод.
  • Чистая прибыль — расчётное поле (выручка − расходы), не хранится отдельно, считается на лету в API за выбранный период.
  • Мобильная версия — KPI сворачиваются в вертикальный стек, таблицы становятся карточками. Календарь на экране ≤ 640px заменяется списком «ближайшие 7 дней».
