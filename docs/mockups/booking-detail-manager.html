Карточка брони — взгляд руководителя

Текущая /bookings/[id] показывает много данных, но без явной иерархии: оборудование, финансы, скан-сессии и журнал лежат в одном ряду карточек, а ключевые действия (PDF, XML, печать, оплата) прячутся в правой колонке. Этот мокап перестраивает страницу под сценарий руководителя: открыл → понял суть за 3 секунды → распечатал PDF/выгрузил XML → одобрил.

Что добавлено vs текущей страницы

  • Hero-блок с большой суммой, статусом, продолжительностью и квик-метриками.
  • Action bar — PDF Смета / PDF Счёт / PDF Акт / XML 1С / Печать всегда видны.
  • Прибыльность брони NEW — gross, себестоимость, маржа.
  • Логистика NEW — время выдачи/возврата, адрес, контакт.
  • Команда NEW — гаффер, осветители, ставки (из Gaffer CRM).
  • Профиль клиента NEW — броней всего, текущий долг, повторность.
  • Share-link NEW — публичная read-only ссылка для клиента.
  • Конфликты по календарю NEW — если оборудование занято где-то ещё.
  • Print view — отдельная вёрстка под печать (нижняя сцена).

Что переиспользует существующий API

  • GET /api/bookings/:id/invoice.pdf — счёт
  • GET /api/bookings/:id/act.pdf — акт (после возврата)
  • GET /api/bookings/:id/full-estimate/export/pdf — смета с транспортом
  • GET /api/bookings/:id/full-estimate/export/xlsx — то же в Excel
  • GET /api/bookings/:id/full-estimate.xml — XML для 1С/учётки уже есть
  • POST /api/bookings/:id/approve / /reject

Новый бекенд (минимум)

  • GET /api/bookings/:id/profitability — gross/cost/margin
  • GET /api/bookings/:id/conflicts — list of overlapping bookings
  • GET /api/clients/:id/snapshot — total/debt/repeat
  • POST /api/bookings/:id/share-link — публичный токен

Идеи функционала · что обсудить

  1. Profitability snapshot · отдельная карточка с выручкой / себестоимостью / маржой. Считаем себестоимость как амортизация (за-смену-стоимость единицы из карточки оборудования) + смены команды + транспорт + связанные ремонты. Сравниваем с медианой по клиенту/проекту. Новый эндпоинт GET /api/bookings/:id/profitability.
  2. Конфликт по календарю прямо в карточке · оранжевый алёрт сверху, если на даты брони какой-то юнит уже занят в другой брони. Клик — открывается календарь с подсветкой пересечения. Эндпоинт GET /api/bookings/:id/conflicts.
  3. Команда / Crew block · интеграция с Gaffer CRM. Назначаем гаффера и осветителей прямо из карточки, по выбору пересчитывается сумма команды (текущий калькулятор уже в packages/shared/crewCalculator). Полезно: моментально вижу, сколько уйдёт команде, и кто за это берётся.
  4. Логистика блока «Выдача / Доставка / Возврат» · разделили три события (со склада, на площадку, обратно). На карте — точка доставки + контакт водителя. Часто клиент звонит «когда привезут?» — теперь ответ в одной строке.
  5. Share-link для клиента · одна кнопка → публичный read-only URL (HMAC-токен, expiry = endDate + 7 дней). Клиент видит свою бронь и счёт без авторизации. Решает кейс «вы можете прислать смету в Telegram?» одним кликом.
  6. Архив всех файлов одним кликом · кнопка «📥 Архив .zip» собирает PDF Сметы + PDF Счёта + XLSX + XML и отдаёт одним zip с осмысленным именем (booking-2026-05-23-rodomski.zip). Бухгалтерия / клиент / 1С — всем нужно разное, дать все варианты сразу.
  7. Дублировать бронь · кнопка в action-bar. Для повторных клиентов: создаётся черновик с тем же оборудованием/командой/машиной, осталось поменять даты. Сейчас приходится пересоздавать вручную.
  8. Quick-edit оборудования inline · хочу из карточки руководителя в ОДИН клик ткнуть в позицию и сменить кол-во или удалить. Сейчас приходится переходить в режим редактирования брони целиком.
  9. Доступность 4/5 в строке оборудования · точка-индикатор справа: зелёная (всё свободно), жёлтая (на пределе), красная (overbook). Помогает увидеть риск до выдачи.
  10. Чек-лист до выдачи · в логистике пунктом «есть подпись клиента / есть копия паспорта / собрано ли всё». До «Подтверждено» → серое, на ISSUED → зелёное.
  11. Журнал правок · в timeline видны все изменения сметы: «+2 Aputure 400x, −1 софтбокс, +1 200 ₽» с автором и временем. Сейчас это лежит в /admin/audit, но к карточке привязки нет.
  12. Профиль клиента из карточки · 5 броней / 382к оборот / 0 ₽ долг — три цифры решают «брать ли в работу при заявке».
