Логика секции

  • Вся секция — только для руководителя. Кладовщик и техник не видят /finance* ни в меню, ни по прямой ссылке (403). Закрепим в матрице прав.
  • Aging-бакеты на странице долгов (>30 / 1–30 / 1–7 / в графике) — докручиваем computeDebts() в apps/api/src/services/finance.ts.
  • Новая Prisma-модель Payment (id, bookingId, amount, receivedAt, method, note?). Календарь и две колонки «ожидается/поступило» строятся агрегацией по дате.
  • Новая Prisma-модель Expense (id, date, category enum, amount, description, documentUrl?, linkedBookingId?). Категории фиксированные: REPAIR, RENT, PURCHASE, PAYROLL, OTHER. Лейблы в UI — по-русски.
  • Связь с ремонтом. Когда в repair-workflow карточка ремонта закрывается с платной работой — автоматически предлагается создать Expense{category: REPAIR, linkedBookingId}. Один ввод, связанность сохраняется.
  • Градиентный хайлайт (blue → purple → orange, наклон −1.5°) используется один раз на странице — как якорь внимания. На сводке — чистая прибыль, на долгах — самый критичный должник, на поступлениях — крупнейший ожидаемый платёж, на расходах — аномально выросшая категория.
  • Мобильная версия. На ≤ 640px aging-бар сворачивается в 2×2, календарь превращается в список «ближайшие 7 дней», таблицы становятся карточками. Угловые квадраты и сетка остаются — они работают и в мобиле.
