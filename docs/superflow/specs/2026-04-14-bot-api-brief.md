# Product Brief: Bot API & Documentation

## Problem Statement
Владелец рентала хочет управлять системой (бронями, финансами) через Telegram-бота на OpenAI, а не через веб-интерфейс. Текущий REST API готов на 80%, но не задокументирован для внешнего потребителя, и нет агрегированных финансовых свёрток («кто должен и сколько»).

## Jobs to be Done
- Когда клиент звонит и хочет забронировать оборудование, я диктую боту в Telegram и бронь создаётся без открытия ноута
- Когда мне надо понять, кто должен денег, я спрашиваю бота и получаю список со сколько и сколько дней просрочено
- Когда я даю указание боту «перенеси бронь X на завтра», он изменяет её в системе без моего клика

## User Stories
1. Как владелец, я пишу боту «создай бронь для Ромашки на камеру Sony с 20 по 22 апреля» → бот через API ищет клиента, подбирает оборудование, создаёт DRAFT-бронь → показывает мне что собирается записать → я подтверждаю → бронь в CONFIRMED
2. Как владелец, я спрашиваю бота «кто мне должен» → получаю список из 5 клиентов с суммами и днями просрочки
3. Как владелец, я пишу «перенеси бронь #42 на три дня позже» → бот обновляет даты в системе
4. Как разработчик, я читаю `docs/bot-api.md` и за 10 минут собираю OpenAI-тулзы по готовым JSON-схемам

## Success Criteria
- Bot-ключ с префиксом `openclaw-` имеет доступ только к whitelist роутов; DELETE/админские роуты → 403
- `GET /api/finance/debts` возвращает агрегацию за < 500 мс на текущей БД
- `docs/bot-api.md` содержит готовые JSON-схемы для OpenAI function-calling, которые можно скопировать в код бота без правок
- `dryRun: true` на POST/PATCH /api/bookings возвращает payload, который БЫ был записан, но без записи в БД
- Написаны тесты на новый middleware (scope guard) + debts endpoint + dryRun (≥ 5 тест-кейсов)

## Scope

### In Scope
- Новый эндпоинт `GET /api/finance/debts` (агрегация + разбивка по проектам)
- Middleware `botScopeGuard`: ограничение прав для ключей с префиксом `openclaw-`
- Параметр `dryRun: true` для `POST /api/bookings` и `PATCH /api/bookings/:id`
- Документация `docs/bot-api.md`: auth, эндпоинты, примеры, ошибки
- JSON-схемы OpenAI function-calling в `docs/bot-api-tools.json`
- Тесты на новый функционал

### Out of Scope
- DELETE любых сущностей через бот-ключ
- Управление учётками админов через API
- OpenAPI-спека / Swagger UI
- MCP-сервер
- Финансовые отчёты кроме долгов (прибыль, cashflow, топ клиентов) — отдельная будущая фича
- Реализация самого бота (только контракт и доки)

## Edge Cases
- Клиент с 0 бронями → не попадает в debts
- Бронь со статусом CANCELLED → не попадает в debts (отменённые не считаются долгом)
- Бронь, где `paidAmount > totalAmount` → переплата, не долг (игнорируем)
- `dryRun: true` + невалидные даты → возвращаем 422 без БД-записи
- Ключ `openclaw-*` пытается DELETE → 403 с `{ message: "Bot keys are not allowed to delete" }`

## Delivery Order
- Sprint 1 (single): всё выше в одном PR. Light-mode governance (1 технический ревьюер).
