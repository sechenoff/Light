# Модель парка техники (как в смете SvetoBaza)

В Excel сметы используются колонки **в таком порядке**:

| Порядок | Заголовок в таблице      | Поле в БД (`Equipment`)   |
|--------:|--------------------------|---------------------------|
| 1       | Перечень оборудования    | `name`                    |
| 2       | Кол-во                   | `totalQuantity`           |
| 3       | Стоимость                | `rentalRatePerShift`      |

Дополнительно в БД (не дублируют лишние колонки сметы):

- `category` — группа/раздел (в файле часто идёт отдельной строкой без кол-ва и цены).
- `importKey`, `brand`, `model`, `comment`, режим учёта `stockTrackingMode` и т.д.

**Не храним** отдельными полями (как в удалённых колонках Excel):

- Общая стоимость — считается как \(Кол\text{-}во \times Стоимость\) при необходимости.
- Заказ Количество — это количество в конкретной брони (`BookingItem.quantity`).
- Сумма аренды — итог по брони/смете (`Estimate` / расчёт на сервере).

Импорт из Excel подхватывает заголовки **«Перечень оборудования»**, **«Кол-во»**, **«Стоимость»** и не использует колонки «Общая стоимость», «Заказ Количество», «Сумма аренды» для записи в парк.

**Период брони:** в `Booking.startDate` / `endDate` хранится **точное дата-время** выдачи и возврата (UTC в БД). Для устаревших запросов допускается только дата `YYYY-MM-DD`: начало — 00:00 UTC, конец — 23:59:59.999 UTC этого дня.

**Смена:** **24 часа** непрерывной аренды. Число биллируемых смен = \(\lceil \Delta t / 24\text{ ч} \rceil\), минимум 1 при \(\Delta t > 0\).

**Расчёт строки в брони:** `BookingItem.quantity × Equipment.rentalRatePerShift ×` (число смен по интервалу `startDate`–`endDate`). Поля `rentalRateTwoShifts` / `rentalRatePerProject` в котировке не используются.
